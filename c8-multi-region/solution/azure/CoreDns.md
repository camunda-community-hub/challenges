# AKS Cross-Cluster DNS SetupThis guide is the AKS equivalent of the GKE [KubeDns.md](../kubedns/KubeDns.md) setup. It walks through enabling cross-cluster DNS resolution between two AKS clusters using CoreDNS, required for Camunda Zeebe multi-region deployment.## Files in This Directory| File | Cluster | GKE Equivalent ||---|---|---|| `coredns-proxy-east.yaml` | East | `dns-lb.yaml` || `coredns-proxy-west.yaml` | West | `dns-lb.yaml` || `coredns-custom-east.yaml` | East | `kubectl edit configmap kube-dns` || `coredns-custom-west.yaml` | West | `kubectl edit configmap kube-dns` || `zeebegateway-loadbalancer.yaml` | Both | `zeebegateway-loadbalancer.yaml` (unchanged) |## Clusters| Name | Context | Region | Namespace ||---|---|---|---|| East | `joey_k8s` | East US | `green-east` || West | `joey_k8s_westus2` | West US 2 | `blue-west` |---## Step 1 — Create NamespacesEach cluster needs a namespace that matches the domain name used in Camunda's contact points.```shell# East clusterkubectl config use-context joey_k8skubectl create namespace green-east# West clusterkubectl config use-context joey_k8s_westus2kubectl create namespace blue-west```---## Step 2 — Deploy the CoreDNS Proxy (replaces dns-lb.yaml)In GKE, `dns-lb.yaml` exposed `kube-dns` externally. In AKS, CoreDNS does not resolve internal service names for queries coming from outside the cluster, so we deploy a dedicated CoreDNS proxy pod on each cluster with a public LoadBalancer instead.### On East Cluster```shellkubectl config use-context joey_k8skubectl apply -f coredns-proxy-east.yamlkubectl apply -f zeebegateway-loadbalancer.yaml -n kube-system```### On West Cluster```shellkubectl config use-context joey_k8s_westus2kubectl apply -f coredns-proxy-west.yamlkubectl apply -f zeebegateway-loadbalancer.yaml -n kube-system```### Get the Proxy LoadBalancer IPsWait for both services to get their external IPs:```shell# East proxy LB IPkubectl config use-context joey_k8skubectl get svc coredns-proxy-lb -n kube-system# West proxy LB IPkubectl config use-context joey_k8s_westus2kubectl get svc coredns-proxy-lb -n kube-system```Example output:```NAME               TYPE           CLUSTER-IP     EXTERNAL-IP       PORT(S)        AGEcoredns-proxy-lb   LoadBalancer   10.0.214.207   172.212.39.11     53:32374/UDP   23s  ← East proxy IPcoredns-proxy-lb   LoadBalancer   10.0.242.167   172.193.220.146   53:30837/UDP   23s  ← West proxy IP```**Record these IPs — you will use them in Steps 3 and 4.**| Cluster | Proxy LB IP ||---|---|| East (`joey_k8s`) | `172.212.39.11` || West (`joey_k8s_westus2`) | `172.193.220.146` |---## Step 3 — Configure CoreDNS Forwarding (replaces kube-dns stubDomains)In GKE, you edited the `kube-dns` ConfigMap to add `stubDomains`. In AKS, we apply `coredns-custom-east.yaml` and `coredns-custom-west.yaml` instead.> **Important:** If your proxy LB IPs from Step 2 differ from the values above, update the `forward` lines in both files before applying.### On East ClusterThe east cluster forwards all `blue-west.*` and `green-east.*` queries to the **west proxy IP** (`172.193.220.146`).```shellkubectl config use-context joey_k8skubectl apply -f coredns-custom-east.yamlkubectl rollout restart deployment coredns -n kube-system```### On West ClusterThe west cluster forwards all `green-east.*` and `blue-west.*` queries to the **east proxy IP** (`172.212.39.11`).```shellkubectl config use-context joey_k8s_westus2kubectl apply -f coredns-custom-west.yamlkubectl rollout restart deployment coredns -n kube-system```---## Step 4 — Create NSG RulesAzure NSG rules are required to allow DNS traffic (UDP port 53) to flow between the two clusters. There is no GKE equivalent — this is Azure-specific.### Get the Cluster Outbound IPsEach cluster has a public outbound IP used when its nodes make external requests. Run these commands to find them:```shell# East cluster outbound IPaz network public-ip show \  --ids $(az aks show --resource-group joey_k8s-rg --name joey_k8s \    --query "networkProfile.loadBalancerProfile.effectiveOutboundIPs[0].id" -o tsv) \  --query ipAddress -o tsv# → 20.121.239.123# West cluster outbound IPaz network public-ip show \  --ids $(az aks show --resource-group joey_k8s_westus2-rg --name joey_k8s_westus2 \    --query "networkProfile.loadBalancerProfile.effectiveOutboundIPs[0].id" -o tsv) \  --query ipAddress -o tsv# → 20.190.21.172```### IP Reference Table| Value | IP | Where it comes from ||---|---|---|| East proxy LB IP | `172.212.39.11` | Step 2 — east `coredns-proxy-lb` EXTERNAL-IP || West proxy LB IP | `172.193.220.146` | Step 2 — west `coredns-proxy-lb` EXTERNAL-IP || East cluster outbound IP | `20.121.239.123` | `az network public-ip show` on east LB || West cluster outbound IP | `20.190.21.172` | `az network public-ip show` on west LB |---### East Cluster NSG: `aks-agentpool-24357108-nsg`Resource Group: `mc_joey_k8s-rg_joey_k8s_eastus`#### Inbound — Allow general traffic between clusters```shellaz network nsg rule create \  --resource-group mc_joey_k8s-rg_joey_k8s_eastus \  --nsg-name aks-agentpool-24357108-nsg \  --name bluegreencommunication \  --priority 999 \  --direction Inbound \  --source-address-prefixes '*' \  --destination-address-prefixes '*' \  --access Allow \  --protocol '*' \  --source-port-ranges '*' \  --destination-port-ranges '*'```#### Inbound — Allow DNS from West clusterThe west cluster (outbound IP `20.190.21.172`) sends DNS queries to the east proxy LB (`172.212.39.11`) on UDP port 53.```shellaz network nsg rule create \  --resource-group mc_joey_k8s-rg_joey_k8s_eastus \  --nsg-name aks-agentpool-24357108-nsg \  --name allowDNSfromWest \  --priority 997 \  --direction Inbound \  --source-address-prefixes 20.190.21.172 \  --destination-address-prefixes 172.212.39.11 \  --destination-port-ranges 53 \  --protocol Udp \  --access Allow```#### Outbound — Allow DNS to West clusterEast cluster nodes send DNS queries to the west proxy LB (`172.193.220.146`) on UDP port 53.```shellaz network nsg rule create \  --resource-group mc_joey_k8s-rg_joey_k8s_eastus \  --nsg-name aks-agentpool-24357108-nsg \  --name allowDNStoWest \  --priority 998 \  --direction Outbound \  --source-address-prefixes '*' \  --destination-address-prefixes 172.193.220.146 \  --destination-port-ranges 53 \  --protocol Udp \  --access Allow```---### West Cluster NSG: `aks-agentpool-25057050-nsg`Resource Group: `mc_joey_k8s_westus2-rg_joey_k8s_westus2_westus2`#### Inbound — Allow general traffic between clusters```shellaz network nsg rule create \  --resource-group mc_joey_k8s_westus2-rg_joey_k8s_westus2_westus2 \  --nsg-name aks-agentpool-25057050-nsg \  --name bluegreencommunication \  --priority 999 \  --direction Inbound \  --source-address-prefixes '*' \  --destination-address-prefixes '*' \  --access Allow \  --protocol '*' \  --source-port-ranges '*' \  --destination-port-ranges '*'```#### Inbound — Allow DNS from East clusterThe east cluster (outbound IP `20.121.239.123`) sends DNS queries to the west proxy LB (`172.193.220.146`) on UDP port 53.```shellaz network nsg rule create \  --resource-group mc_joey_k8s_westus2-rg_joey_k8s_westus2_westus2 \  --nsg-name aks-agentpool-25057050-nsg \  --name allowDNSfromEast \  --priority 997 \  --direction Inbound \  --source-address-prefixes 20.121.239.123 \  --destination-address-prefixes 172.193.220.146 \  --destination-port-ranges 53 \  --protocol Udp \  --access Allow```#### Outbound — Allow DNS to East clusterWest cluster nodes send DNS queries to the east proxy LB (`172.212.39.11`) on UDP port 53.```shellaz network nsg rule create \  --resource-group mc_joey_k8s_westus2-rg_joey_k8s_westus2_westus2 \  --nsg-name aks-agentpool-25057050-nsg \  --name allowDNStoEast \  --priority 998 \  --direction Outbound \ DNS resolution using busybox.### From East — resolve a West service```shellkubectl config use-context joey_k8skubectl run dns-test --rm -it --image=busybox -n default -- sh# Inside shell:nslookup test-service.blue-west.svc.cluster.local# Expected: resolves to a ClusterIP in the west cluster (e.g. 10.0.250.35)```### From West — resolve an East service```shellkubectl config use-context joey_k8s_westus2kubectl run dns-test --rm -it --image=busybox -n default -- sh# Inside shell:nslookup test-service.green-east.svc.cluster.local# Expected: resolves to a ClusterIP in the east cluster (e.g. 10.0.36.22)```---## Camunda Zeebe Contact PointsOnce Camunda is deployed in the `green-east` namespace on east and `blue-west` namespace on west, configure the Zeebe broker with the following contact points:```yaml- name: ZEEBE_BROKER_CLUSTER_INITIALCONTACTPOINTS  value: "camunda-zeebe-0.camunda-zeebe.green-east.svc.cluster.local:26502,          camunda-zeebe-1.camunda-zeebe.green-east.svc.cluster.local:26502,          camunda-zeebe-0.camunda-zeebe.blue-west.svc.cluster.local:26502,          camunda-zeebe-1.camunda-zeebe.blue-west.svc.cluster.local:26502"```
